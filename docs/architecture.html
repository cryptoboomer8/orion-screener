Orion Screener — Pipeline Architecture

Every 15 minutes, a Cloudflare Worker fetches the full screener payload, gzips it, and lands it in GitHub. From there a workflow re-aggregates the lean dashboard data, GitHub Pages redeploys, and the static site picks it up on the next page load. Click any node for the underlying file, command, or schedule.

Why each layer is separable. Each block speaks to the next through a stable contract — the gzipped file in the repo. Swap the Worker for Lambda or Render, swap the aggregator for a Worker, swap Pages for Cloudflare Pages: as long as the file format and path stay the same, nothing else has to change.

Cron schedule

Set in worker/wrangler.toml. Workers cron has sub-minute precision and is free at this volume.

[triggers] crons = ["*/15 * * * *"]

Worker code

One TypeScript file, ~120 lines. Exposes both a scheduled handler and a fetch handler so you can curl it for a manual snapshot.

worker/src/index.ts

Manual run:

https://orion-snapshot.<subdomain>.workers.dev/

Upstream API

Public, no auth required. Cloudflare-fronted, which is why the Worker had to live inside Cloudflare — data-center IPs (Render, AWS) get 403'd by bot protection.

GET https://screener.orionterminal.com/api/screener

Returns { tickers: […], lastUpdate, symbolCount, version, type }.

GitHub commit

The Worker writes via the Contents API using a fine-grained PAT scoped to this repo with contents:write only.

PUT /repos/cryptoboomer8/orion-screener/contents/snapshots/<file>

Token is stored as a Worker secret — never in the repo.

Snapshot file

Wrapped payload, gzipped. Filename uses UTC for new files (legacy ones use CEST — the dashboard sorts by parsed timestamp so ordering is correct either way).

{ "timestamp": "2026-04-15T20:30:00+00:00", "screener": { "tickers": […], "lastUpdate": … } }

Repo growth

~5 MB/day at the new cadence. Comfortable for ~3 months, then migration to Cloudflare R2 starts to make sense.

github.com/cryptoboomer8/orion-screener

Workflow

Path-filtered to snapshots/** so it only fires when the Worker pushes. Skips the commit if nothing changed.

.github/workflows/aggregate-dashboard.yml

Aggregator

Handles both file formats, ranks by 5-minute trade count to match the original agent, sorts by parsed ISO timestamp so legacy CEST and new UTC files interleave correctly.

scripts/aggregate_snapshots.py

Output row shape: { ts, tR: [20 ints], vR: [20 ints] }.

Lean dashboard data

The dashboard sums tR[0:N] and vR[0:N] live based on the Top-N slider, so per-rank arrays are kept rather than pre-aggregated totals.

docs/data/snapshots.json

Hosting

GitHub Pages, branch + folder source. Automatic redeploy on push, no build step. Free, ample bandwidth.

https://cryptoboomer8.github.io/orion-screener/

Dashboard frontend

Single self-contained HTML with inline CSS/JS plus Chart.js from CDN. No build pipeline. Cache-busted fetch on every load.

fetch('./data/snapshots.json?_=' + Date.now())

What you see

Top-N slider (1–20), Mean/Median toggle, heatmap with percentile-clipped color scale, hourly chart with session bands and IQR error bars, Session Brief (today vs typical pace + next 5 hours), and three KPIs: Best Hour, Best 2h Window, Best Day.
